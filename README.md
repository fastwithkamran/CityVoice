# 🏙️ CityVoice: Urban Planning Platform

**Your City, Your Voice.**

### 🌐 [Live Demo](https://cityvoice-two.vercel.app/)

CityVoice is a modern, real-time platform designed to bridge the gap between citizens and urban planners. It empowers residents to actively participate in the development of their neighborhoods by sharing feedback, proposing ideas, and prioritizing urban projects through a democratic voting system.

---

## 🚀 The "Why" behind CityVoice

Urban development often happens behind closed doors, leading to a disconnect between infrastructure projects and the actual needs of the people. CityVoice was built to:
- **Democratize Planning**: Give every citizen a seat at the table.
- **Identify Pain Points**: Surface localized problems (like broken lighting or drainage issues) that planners might miss.
- **Spark Innovation**: Provide a space for creative urban improvements and "Smart City" ideas.
- **Data-Driven Decisions**: Provide planners with real-time statistics and upvote data to help prioritize high-impact projects.

## 🎯 Core Focus

The platform focuses on **Actionable Transparency**. By categorizing every submission and allowing the community to vote, the most urgent and popular needs naturally rise to the top, creating a clear roadmap for urban improvement.

---

## 🛠️ Tech Stack

This application is built with a modern, high-performance stack:
- **React**: For a dynamic and responsive user interface.
- **Vite**: For lightning-fast builds and development.
- **Firebase**: Leveraging Firestore for real-time data persistence and upvoting synchronization.

## 🔑 Environment Setup

To run this project locally or deploy your own instance, you need a Firebase project. Create a `.env` file in the root directory with the following keys:

```env
VITE_FIREBASE_API_KEY=your_api_key
VITE_FIREBASE_AUTH_DOMAIN=your_auth_domain
VITE_FIREBASE_PROJECT_ID=your_project_id
VITE_FIREBASE_STORAGE_BUCKET=your_storage_bucket
VITE_FIREBASE_MESSAGING_SENDER_ID=your_messaging_sender_id
VITE_FIREBASE_APP_ID=your_app_id
VITE_FIREBASE_MEASUREMENT_ID=your_measurement_id
```

## 💻 Development

Follow these steps to get a local development server running:

1.  **Install Dependencies**:
    ```bash
    npm install
    ```
2.  **Start Dev Server**:
    ```bash
    npm run dev
    ```
3.  **Note**: Ensure your `.env` file is configured as described in the **Environment Setup** section to connect to the database.
